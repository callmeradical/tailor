Requirements

Tailor targets macOS. Linux works for most roles. Windows is out of scope.

  • macOS 12 Monterey or later (earlier may work, untested)
  • Xcode Command Line Tools (xcode-select --install)
  • Ansible 2.14+ (pip3 install ansible or brew install ansible)
  • community.general collection (ansible-galaxy collection install community.general)

The bootstrap script handles all of this automatically.

Bootstrap

On a fresh machine, run one command:

curl -fsSL https://raw.githubusercontent.com/callmeradical/tailor/main/install.sh \
  | bash -s -- https://github.com/yourname/tailor-config.git

This will:

  1. Install Xcode Command Line Tools (if missing)
  2. Install Homebrew (if missing)
  3. Install Ansible via pip3
  4. Install the community.general Ansible collection
  5. Clone your config repo to ~/.config/tailor
  6. Run tailor apply
Idempotent. Safe to re-run on an already-configured machine. Only drift gets fixed.

You can override the config directory:

TAILOR_CONFIG_DIR=~/my-config curl -fsSL ... | bash -s -- <repo-url>

First apply

Once bootstrapped, apply at any time:

# Dry-run first — see what would change
tailor apply --check

# Apply for real
tailor apply

Repo structure

Your config repo is a standard Ansible project. Fork the starter or create your own:

~/.config/tailor/
  site.yml             # main playbook — lists all roles
  group_vars/
    all.yml            # your config: packages, dotfiles url, prefs
  requirements.yml     # ansible-galaxy collection deps
  roles/
    packages/          # homebrew formulae + casks
    dotfiles/          # dotfiles repo clone + installer
    macos-prefs/       # osx_defaults tasks
    vscode/            # vs code extensions
    hooks/             # pre/post apply shell scripts

You can add your own roles, pull from Ansible Galaxy, or write custom tasks directly in site.yml. It's just Ansible.

group_vars/all.yml

This is your primary config file. All roles read their variables from here.

# Packages
homebrew_packages:
  - git
  - ripgrep
  - fd
  - fzf
  - gh

homebrew_casks:
  - iterm2
  - visual-studio-code
  - 1password

# Dotfiles
dotfiles_repo_url: "https://github.com/yourname/dotfiles.git"
dotfiles_dest: "{{ ansible_facts['env']['HOME'] }}/dotfiles"
dotfiles_installer: "install.sh"  # set to "" to skip

# macOS preferences (override role defaults)
macos_dock_autohide: true
macos_dock_tilesize: 48
macos_key_repeat: 2
macos_initial_key_repeat: 15
macos_trackpad_tap_to_click: true

# VS Code extensions
vscode_extensions:
  - vscodevim.vim
  - eamodio.gitlens
  - esbenp.prettier-vscode

Packages

The packages role installs Homebrew formulae, casks, and taps.

homebrew_packages:       # brew install
  - git
  - ripgrep

homebrew_casks:          # brew install --cask
  - visual-studio-code

homebrew_taps:           # brew tap
  - homebrew/cask-fonts

To add a package you installed manually:

tailor adopt brew ripgrep

Dotfiles

The dotfiles role clones your existing dotfiles repo and runs its installer. It does not manage dotfile content — it integrates with whatever setup you already have.

dotfiles_repo_url: "https://github.com/yourname/dotfiles.git"
dotfiles_dest: "{{ ansible_facts['env']['HOME'] }}/dotfiles"
dotfiles_installer: "install.sh"  # relative to dotfiles_dest, "" to skip

Leave dotfiles_repo_url empty ("") to skip the role entirely.

To register an existing config file:

tailor adopt file ~/.gitconfig

This copies the file into roles/dotfiles/files/ and adds a symlink task. Files starting with . are linked to ~/<filename>; others to ~/.config/<filename>.

macOS preferences

The macos-prefs role applies system defaults using Ansible's osx_defaults module. Override any default in group_vars/all.yml.

VariableDefaultDescription
macos_dock_autohidetrueAuto-hide the Dock
macos_dock_tilesize48Dock icon size (px)
macos_key_repeat2Key repeat rate (1=fastest)
macos_initial_key_repeat15Delay before repeat
macos_trackpad_tap_to_clicktrueTap to click
macos_screenshot_location~/DesktopScreenshot save location

Apply only this role:

tailor apply --role macos-prefs

VS Code

The vscode role installs extensions by their Marketplace ID. Skips gracefully if code is not on $PATH.

vscode_extensions:
  - vscodevim.vim
  - eamodio.gitlens
  - ms-python.python

To get IDs for your current extensions:

code --list-extensions

Hooks

Place shell scripts in roles/hooks/ to run before or after the apply:

roles/hooks/pre-apply.sh   # runs before all roles
roles/hooks/post-apply.sh  # runs after all roles

Both are no-ops if absent. Use hooks to sign in to 1Password, source secrets, or run anything Ansible doesn't cover.

#!/bin/sh
# roles/hooks/pre-apply.sh — example: sign in to 1Password
eval $(op signin)

tailor apply

tailor apply [--check] [--role <name>] [--verbose]
FlagDescription
--checkDry-run. Shows what would change. No system modifications.
--role <name>Apply a single role by tag. e.g. packages, macos-prefs
--verbosePass -v to Ansible for verbose output.

tailor adopt

tailor adopt brew <package>
tailor adopt file <path>

Captures a single ad-hoc change back into the declared config. Always opens the modified file in $EDITOR for review. Idempotent — running twice has no effect.

Workflow: Install something manually → adopt it → commit the change to git.

tailor validate

tailor validate

Syntax-checks the playbook using ansible-playbook --syntax-check. Parse-only — no task evaluation, no system changes. Use before applying after editing roles.

tailor --usage

tailor --usage

Prints a structured agent-oriented briefing: config location, key variables, role reference, and workflow. Designed to be consumed by AI agents and automation tools at session start.

# Example: orient an agent
tailor --usage | head -40

Guide: New machine

  1. Open a terminal.
  2. Run the bootstrap command with your config repo URL.
  3. Wait. Tailor installs Ansible, clones your config, and applies it.
  4. Source your shell environment if your dotfiles installer added anything to ~/.zshrc.
  5. Done.
curl -fsSL https://raw.githubusercontent.com/callmeradical/tailor/main/install.sh \
  | bash -s -- https://github.com/yourname/tailor-config.git

Guide: Daily workflow

# Installed something you want to keep? Adopt it.
brew install ripgrep
tailor adopt brew ripgrep
cd ~/.config/tailor && git add -A && git commit -m "adopt: ripgrep"

# Edited a config file? Version-control it.
tailor adopt file ~/.gitconfig
cd ~/.config/tailor && git add -A && git commit -m "adopt: .gitconfig"

# Changed prefs? Update group_vars/all.yml and apply.
# Edit ~/.config/tailor/group_vars/all.yml ...
tailor apply --role macos-prefs

# Keep two machines in sync.
cd ~/.config/tailor && git pull
tailor apply

Guide: Using with AI agents

Tailor ships a machine-readable briefing command for AI agents and automation tools.

# Orient an agent at session start
tailor --usage

The output covers: config file location, all key variables with current defaults, role reference, adopt workflow, and which commands are safe to run without system changes.

Safe commands (no system modifications):

  • tailor apply --check — dry-run diff
  • tailor validate — syntax check only
  • tailor --usage — this briefing
  • cat ~/.config/tailor/group_vars/all.yml — read current config

Commands that modify the system:

  • tailor apply — installs packages, applies prefs
  • tailor adopt brew <pkg> — writes to config file
  • tailor adopt file <path> — copies file, writes to config
